# 验收 · 质量门禁

> 每个 PR / 里程碑都必须穿过对应门禁,否则不算完成。门禁 80% 自动化,20% 人工 sign-off。

## 1. PR 级门禁(自动)

每个 PR 必须全绿才能合并:

| 检查 | 工具 | 阈值 |
|------|------|------|
| Lint | ESLint / Checkstyle / ruff | 0 error |
| 单测 | Vitest / JUnit / pytest | 覆盖率 ≥ 70%、关键模块 ≥ 85% |
| 类型 | tsc / mypy | 0 error |
| 契约一致性 | spectral + oasdiff + buf breaking + jsonschema | 无 breaking 或同 PR 同步改 |
| 安全扫描 | Trivy / Snyk / GitGuardian | 0 high+,无密钥 |
| 镜像签名 | cosign | 必有 |
| 二进制大小 | size-limit(前端) | 主 chunk gzip ≤ 180KB |
| Lighthouse(前端) | LHCI | LCP ≤ 1.5s,A11y ≥ 95(关键页) |
| Storybook 视觉回归(前端) | Chromatic / Loki | 无意外变化 |
| OpenAPI 与代码一致(后端) | swagger-codegen check | diff = 0 |
| 数据库迁移 | flyway dry-run | 可前向可回滚 |

## 2. 服务级门禁(每次发布到 staging)

| 检查 | 阈值 |
|------|------|
| 启动健康 | 30s 内 Ready |
| Smoke 测试 | golden path 100% |
| 集成测试 | Pact contract 全过 |
| Trace / Log / Metric 接入 | 100% |
| Dashboard | 已部署 + 至少一条 alert |
| Runbook | 服务 README + RUNBOOK 存在 |

## 4. 自动化触发(Orchestrator 视角)

```
PR 开 → 自动 PR 门禁
        通过 → 合并到 main
        失败 → 退回 Owner
合并到 main → 部 staging → 服务级门禁
        通过 → 标 release-candidate
里程碑结束 → 触发 QA Agent 跑里程碑 DoD
        通过 → Orchestrator 解锁下一里程碑
        失败 → 列阻塞清单 → 重派
```
