# 风险与并行策略

## 1. 关键路径

```
M0 ──► M1 ──► M2 ─┬─► M3 ──► M4 ──► M5 ──► M6 ──► M7 ──► M8
                  └─► M2.5(并行,关键)────────┘
```

**关键路径上的任务**(任一延期 = 整体延期):

1. `packages/proto` 契约冻结(M0 末)
2. gateway-ws 长连接 50k 压测(M1 末)
3. ai-hub 决策器 + 流式输出(M2 中)
4. tool-svc 直播领域工具适配(M2.5)
5. web-c 形态适配(M4)
6. 突发流量演练通过(M6)

## 2. 并行调度建议

| 阶段 | 可并行 | 不可并行(依赖) |
|------|--------|------------------|
| M0 | FE 组件库 / BE 契约 / OPS 平台 / DOC 模板 | - |
| M1 | gateway-ws / session-svc / web-c 静态壳 / web-admin 壳 / notify-svc | upload 之前要 OSS 准备好 |
| M2 | llm-router / kb-svc / FAQ 表 / web-admin AI 配置壳 | ai-hub 要等 router+kb |
| M2.5 | livectx-svc / tool-svc 框架 / sdk-jsbridge | tool 适配要等业务方提供 mock |
| M3 | routing-svc / agent-bff / web-agent | 主管视图要等 report 雏形 |
| M4 | C 端各 Story | 形态适配要等 sdk 完成 |
| M5 | report-svc / 各 dashboard / 质检 | A/B 要等 ai-hub 模板版本化 |
| M6 | Chaos / 压测 / 渗透 / 合规 | 必须等 M5 全绿 |

并行原则: **同 Agent 内串行,跨 Agent 并行**;契约必须先冻结。

## 3. 主要风险与预案

### R1 · 业务方接口不到位(高)
- 现象: 播放诊断 API、CDN webhook、订阅、礼物 任一未提供 → AI 工具调用空跑
- 预案:
  - M0 即与上游团队对齐协议 + 提供 mock-server(`tools/business-mock`)
  - tool-svc 全部工具支持 `mock_mode` 通过环境变量切换
  - 真接口落地前 staging 全走 mock,不阻塞下游

### R2 · LLM 成本失控(高)
- 现象: token 消耗远超预算
- 预案:
  - 默认 `gpt-4o-mini`,只在 RAG 复杂情况切 `gpt-4o`
  - FAQ 命中优先(零 token)
  - 缓存命中率监控,目标 ≥ 30%
  - 每日 token 预算告警 80% / 100%
  - 100% 时强制切便宜档或全人工

### R3 · 长连接稳定性(中)
- 现象: gateway-ws 在大主播开播时连接抖动
- 预案:
  - 提前压测 + Chaos
  - 预备 SSE / 长轮询降级
  - 一致性 hash 减少扩缩容时连接迁移

### R4 · WebView 兼容差异(中)
- 现象: 不同宿主 App 的 JSBridge / 横屏 / PIP 行为差异
- 预案:
  - 在 M0 与最大业务方主播 App 对齐协议
  - 真机矩阵覆盖 iOS WK / Android System × 2 宿主
  - JSBridge 缺失字段一律降级到 polyfill / fullscreen

### R5 · 直播突发流量(高)
- 现象: 大主播开播 / 赛事 5min 内 10x~20x
- 预案:
  - 与运营日历联动预热 HPA
  - 群发模式: 同 room 高并发 → FAQ-only + 工单兜底
  - 一键降级开关(关流式/关建议/RAG-only/人工)

### R6 · 合规 / 未成年退款流程不达标(中)
- 现象: 流程缺双人复核 / 留痕不全 / 时效不达
- 预案:
  - M2 提前与法务对齐 SOP
  - M5 上线时跑合规演练
  - 季度审计 + 季度演练

### R7 · 知识库初始化数据不足(中)
- 现象: M2 上线时 RAG 命中率低
- 预案:
  - M0 启动同步,与运营/客服收集历史 QA、典型工单、节目政策
  - M2 前 1 周完成 ≥ 1k 条入库 + 标注

### R8 · 多 Agent 协作冲突(中)
- 现象: 跨 Agent 改契约导致连锁失败
- 预案:
  - 契约先行,Orchestrator review 拦截 breaking
  - 每个里程碑前 2 天契约冻结日
  - codegen 自动同步,所有 Agent 必须 pull

### R9 · 安全/隐私事故(高)
- 现象: API Key 泄露、live_context 伪造、敏感日志
- 预案:
  - GitGuardian + Trivy + 镜像签名强制门禁
  - 服务端覆盖 live_context 关键字段
  - 日志脱敏中间件强制接入

## 4. 提前介入清单(M0 时即启动)

- 与播放器 SDK 团队对齐 JSBridge 字段、ring buffer
- 与运营/客服收集历史 QA(目标 ≥ 1k 条)
- 与法务/合规对齐未成年/举报/版权 SOP
- 与采购/财务确认 OpenAI 配额、内容审核合作
- 与 SRE 对齐 K8s 集群、CDN 配额、KMS

## 5. 应急预案(线上)

| 事件 | 等级 | 处置 |
|------|------|------|
| LLM 全断 | P0 | 自动转人工模式 + Banner;Token 预算冻结 |
| Gateway 大面积失联 | P0 | 客户端自动重连换节点;运维介入 |
| Mongo 主切失败 | P0 | 切只读模式,新消息走 Redis Stream 暂存 |
| CDN 大故障 | P1 | 公告自动挂、客服群发模式 |
| 合规事件(未成年退款积压) | P1 | 主管接管 + 启用人工绿色通道 |
| 知识库错误回答 | P2 | 紧急下线对应 chunk + Prompt fix |

## 6. Buffer 与浮动

- 每个里程碑预留 20% buffer。
- 风险事项触发 → Orchestrator 优先调度 + 必要时降低本里程碑非关键 Story。
- M6 高可用 + M7 灰度 共享 1 周 buffer,极限情况下 M8 GA 顺延。
