# 契约优先工作流

> 多 Agent 并行的前提:**契约先冻结,实现并行展开**。

## 1. 契约清单

放在 `packages/proto/`,所有 Agent 共享。

```
packages/proto/
├── openapi/
│   ├── visitor.yaml          # C 端 REST
│   ├── agent.yaml            # 座席 BFF REST
│   ├── admin.yaml            # 管理后台 REST
│   ├── ai-internal.yaml      # AI 内部
│   └── tool.yaml             # 工具协议
├── ws/
│   ├── client.schema.json    # 客户端 ↔ 服务端 帧
│   └── events.schema.json    # 系统事件
├── jsbridge/
│   └── kefu.d.ts             # JSBridge TS 类型
├── live-context/
│   └── live-context.schema.json
└── grpc/
    ├── routing.proto
    ├── ai.proto
    └── tool.proto
```

## 2. 工作流

```
[变更人] 改 packages/proto/* → 提 PR
       │
       ▼
[CI] schema lint + breaking-change 检测(oasdiff / buf breaking)
       │
       ▼
[Reviewer] PRD/PM + 受影响 Agent 评审
       │
       ▼
[合并] 触发 codegen → 自动生成
       - TS 类型 (apps/*)
       - Java DTO (services/* spring-doc + openapi-generator)
       - Python pydantic (services/ai-hub, llm-router)
       - Postman collection
       │
       ▼
[Agent] 拉最新 codegen,更新实现
       │
       ▼
[CI] contract-check(运行 pact + schema validation 抽样)
```

## 3. Breaking 与 Non-breaking

| 类型 | 处理 |
|------|------|
| 加字段(可选) | non-breaking,直接合 |
| 删字段 / 改类型 / 改必填 | breaking,需所有受影响 Agent 同步;走 v1 → v1.1 灰度 |
| 加枚举值 | 默认不破坏,但前端需有 default 兜底 |
| WS event 名变更 | breaking,前后端同步,前端兼容期保留旧 handler |

## 4. live_context 协议管控

`live_context` 是直播场景关键契约,需要严格控制版本与字段:

- 当前版本: `v1`,字段见 PRD `02-C端-WebH5-需求.md` §3.6
- **服务端校验**:
  - `uid`、`level`、`is_anchor`、`is_minor_guard` 全部由后端反查覆盖,客户端值仅作 hint
  - `room_id` / `vod_id` / `anchor_id` 与服务端业务系统校验一致(防伪造)
  - 异常字段 → 拒绝 + 记录风控事件
- **演进**:
  - 加字段(如 `device.fingerprint`)→ 走 non-breaking 流程
  - 删/改字段 → 必须双发兼容期 ≥ 2 周

## 5. JSBridge 协议管控

- 命名空间: `kefu.*`
- 版本协商: H5 启动时 `kefu.ready({ minVer: "1.2.0" })`,宿主返回支持版本;不支持则降级形态(全屏 fallback)
- **字段冻结**: `kefu.setLiveContext` 入参 schema 必须等同于 `live-context.schema.json`
- **事件**: `kefu.on(eventName, cb)` 注册;新增事件用 `event_${name}_v2` 避免冲突

## 6. 模拟与 Mock

- `apps/web-c` 开发模式: `VITE_MOCK=true` 启动时 wire 一个 service worker mock,直接读 `packages/proto/openapi` examples
- `services/ai-hub` 单测: 用 `responses-fixtures/` 录回放 LLM 响应,跑 promptfoo
- WS Mock: `tools/ws-mock`(node),按 `ws/events.schema.json` 推 event,前端可单独跑

## 7. 变更冲刺节奏

- 每个里程碑前两天 = **契约冻结日**:`packages/proto` 不再做 breaking
- 里程碑结束后 = **契约清理日**:删除已过兼容期的旧版本

## 8. 最小先行集

M0 必须先冻结的契约:

1. `openapi/visitor.yaml` 的 `auth / sessions / messages / announcements / quick-replies / faq.tree`
2. `ws/client.schema.json` 的 `msg.text / msg.chunk / event.queue_update / event.faq_card`
3. `live-context/live-context.schema.json` v1
4. `jsbridge/kefu.d.ts` 基础方法

其它在 M2 / M2.5 / M3 各阶段陆续 frozen。
