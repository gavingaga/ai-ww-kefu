# AI 执行计划 · 总览

> 版本: v1.0 · 日期: 2026-04-27 · 对应 PRD v1.0
> 目标: 把 PRD 拆成可由「主控 AI Agent + 多个领域 SubAgent」并行/串行执行的工程任务。

## 1. 执行模式

```
            ┌────────────────────────────┐
            │      Orchestrator (主控)      │
            │  读 PRD → 分派 → 验收 → 集成   │
            └──────────┬─────────────────┘
                       │
   ┌───────────────────┼───────────────────┬─────────────────┐
   ▼                   ▼                   ▼                 ▼
┌────────┐       ┌──────────┐       ┌──────────┐     ┌─────────┐
│Frontend│       │ Backend  │       │  AI/RAG   │     │ DevOps  │
│ Agent  │       │  Agent   │       │  Agent    │     │  Agent  │
└────────┘       └──────────┘       └──────────┘     └─────────┘
   │                  │                   │                │
   ▼                  ▼                   ▼                ▼
[C 端 / 后台 / 座席]  [服务/网关/数据]  [LLM/RAG/工具]   [K8s/CI/监控]

            ┌────────────────────────────┐
            │  共享: 协议契约 / 设计令牌    │
            │  (OpenAPI / JSBridge / DT)  │
            └────────────────────────────┘
```

- **Orchestrator**: 持有 PRD + 当前进度,生成 backlog,按里程碑顺序派发给 SubAgent;在每个交付点跑验收 checklist。
- **Frontend Agent**: 输出 React 应用(C 端 / 座席台 / 管理后台)、JSBridge SDK、Storybook。
- **Backend Agent**: 输出微服务(Gateway / Session / Routing / Agent BFF / Notify / Tool / Audit / Report)与 OpenAPI。
- **AI Agent**: 输出 AI-Hub、RAG 流水线、LLM Router、领域工具适配器、Prompt 模板。
- **DevOps Agent**: 输出 IaC、K8s manifests、CI/CD、监控告警、压测脚本。
- **QA Agent**(每个里程碑结束触发): 跑用例、E2E、压测、安全扫描、合规走查。

## 4. 关键路径

```
[M0 准备] ─► [M1 骨架] ─► [M2 AI] ─┬─► [M3 座席台] ─► [M4 C 端完整] ─► [M5 报表] ─► [M6 高可用] ─► [M7 灰度] ─► [M8 GA]
                                  └─► [M2.5 业务集成] ──┘
```

- M2.5 与 M2/M3 并行,**关键路径上**(播放诊断 API 不到位会卡 AI 上下文准确率)。
- M4 C 端完整体验 依赖 M2.5 提供的诊断 API。
- M6 演练 必须在 M4/M5 完成后启动。

## 5. 工作节奏

- **日**: SubAgent 自驱跑 task,主控每日抓进度报表 + 不通过项重派。
- **周**: 跨 Agent 集成(契约对齐 + E2E)、PR review。
- **里程碑**: 触发 QA Agent 全量跑 → 不达 DoD 阻塞下一里程碑。

## 6. 输出物清单

| 类型 | 内容 |
|------|------|
| 代码 | apps / services / packages / infra,所有 PR 必须带测试 |
| 契约 | `packages/proto` 内 OpenAPI 3.1 + WS JSON Schema + JSBridge TS 类型 |
| 文档 | 每个服务 `README.md` + `RUNBOOK.md`(如何跑、如何排障) |
| 自动化 | CI: lint/test/build/sec-scan/contract-check;CD: Argo Rollouts |
| 数据 | 种子数据脚本(默认公告/快捷按钮/FAQ 树/模型档位) |
| 报告 | 每里程碑产出验收报告 + 性能/安全/合规 sign-off |
