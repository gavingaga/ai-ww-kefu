# 阶段与里程碑执行计划

> 与 PRD `10-里程碑与排期.md` 一一对应。每阶段:**目标 → 入口准则(DoR)→ 任务包 → 退出准则(DoD)→ 验收**。

## M0 · 准备 (W1)

**目标**: 立项启动,工程脚手架就绪。

**DoR**
- PRD v1.0 评审通过
- 团队/账号/云资源(K8s, KMS, OSS, CDN, OpenAI 配额)就位

**任务包**
| ID | 任务 | Owner | 输出 |
|----|------|------|------|
| M0-1 | 初始化 monorepo + 包管理(pnpm + Maven + uv) | DevOps | `apps/`、`services/`、`packages/` 空骨架 + Workspace 配置 |
| M0-2 | 设计令牌 + Glassmorphism 组件库 v0 | Frontend | `packages/design-tokens`、`packages/ui-glass` 含 Button/Glass/Avatar/MarqueeBar |
| M0-3 | 契约骨架(OpenAPI / WS / JSBridge) | Backend | `packages/proto` 第一版 schema |
| M0-4 | 通用 CI 流水线 | DevOps | lint/test/build/contract-check + 镜像产出 |
| M0-5 | K8s 基础(命名空间、Ingress、Secrets via KMS、Argo CD) | DevOps | `infra/k8s` Helm chart + 测试集群可部署 hello |
| M0-6 | 可观测基础(Prom/Grafana/Tempo/Loki/OTel SDK) | DevOps | dashboard 模板 + trace 默认 |
| M0-7 | 安全基础(WAF、CSP、CORS、Secret 扫描) | DevOps | 默认 hardening 配置 |
| M0-8 | 视觉走查(Figma 同步) | Frontend | Storybook 跑通 |

**DoD**
- `pnpm i && pnpm -r build` 全绿
- `mvn -T 1C verify` 全绿
- `kubectl apply -k infra/k8s/dev` 部署 hello 服务通,trace/log 全链路可见
- 至少 5 个共享组件在 Storybook 可交互

## M1 · 基础骨架 (W2 ~ W4)

**目标**: 端到端跑通"匿名用户进 → 发文本 → 收文本",含公告 + 快捷按钮 + FAQ 一级。

**DoR**
- M0 全绿
- OpenAPI v0.1 已 frozen(可改但需走变更流程)

**任务包**
| ID | 任务 | Owner | 输出 |
|----|------|------|------|
| M1-1 | gateway-ws: 连接/心跳/路由(Redis 映射)/seq+ack | Backend | Go 服务,单机 50k 连接压测达标 |
| M1-2 | session-svc: 会话状态机 + 消息落库(MongoDB) | Backend | 会话 CRUD、消息幂等、顺序保证 |
| M1-3 | notify-svc: 公告/快捷按钮 CRUD + 推送 | Backend | 后台改动 5s 内 C 端生效 |
| M1-4 | web-c MVP: 玻璃材质 + 消息流 + 输入框 + WS | Frontend | 文本收发 + 公告跑马灯 + 快捷按钮 |
| M1-5 | web-admin MVP: 登录/权限/公告/快捷按钮 | Frontend | 配置即可下发 |
| M1-6 | upload-svc + OSS 直传 | Backend | 临时签名 + 病毒扫描钩子 |
| M1-7 | 默认数据初始化(公告 / 快捷按钮 / 默认模型档位 stub) | DevOps | seeds 脚本 |

**DoD**
- E2E: 用户在 web-c 发送文本 → 后端落库 → 推回 → 历史可查
- 公告/快捷按钮在 web-admin 改动 5s 内 C 端可见
- 单机 gateway-ws 50k 长连接,P99 推送 < 50ms
- 全链路 trace_id 贯通
