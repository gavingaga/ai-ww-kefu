# 非功能性需求

## 1. 性能

| 指标 | 目标 |
|------|------|
| C 端首屏 LCP | ≤ 1.5s (4G) |
| C 端 TTI | ≤ 2.5s |
| 主 chunk gzip | ≤ 180KB |
| WS 连接建立 | ≤ 500ms |
| 消息 E2E P95 | ≤ 300ms |
| AI 首字节 | ≤ 600ms |
| AI 完整回复 P95 | ≤ 2.5s |
| 后台 API P95 | ≤ 200ms |
| 1000 条历史滚动 | ≥ 60fps |

## 2. 可用性

- 系统整体 SLA **99.95%**(月度 ≤ 21min 不可用)。
- 关键链路(发消息 / 接消息)目标 **99.99%**。
- 多 AZ 部署,任一 AZ 故障无感切换。
- 异地热备,RPO ≤ 5min,RTO ≤ 30min。

## 3. 安全

### 3.1 数据安全
- 所有传输 TLS 1.2+,内部 mTLS。
- 静态加密: DB / OSS 静态加密,密钥托管 KMS。
- 个人敏感字段(手机/身份证/银行卡)落库前 AES-GCM 加密 + 单独权限。
- 上传文件病毒扫描(ClamAV / 云服务),通过后才可访问。

### 3.2 应用安全
- OWASP Top 10 全覆盖测试。
- WAF + 限流 + 风控(机器人识别)。
- 严格 CSP、X-Frame-Options、HSTS、CORS 白名单。
- 防越权: 所有 ID 资源走 (tenant_id, owner_id) 作用域校验。
- 文件白名单后缀 + Magic Number 校验,禁止脚本/可执行文件。
- C 端无敏感配置,后台密钥不下发。

### 3.3 账户与权限
- 后台 RBAC,最小权限原则,二步验证(主管/管理员强制)。
- 坐席登录限制并发会话数 / 设备数。
- 审计日志: 配置变更、权限改动、敏感操作 90 天可查。

### 3.4 AI 安全
- Prompt 注入防护: 用户输入隔离 + 系统指令固化;`live_context` 字段强类型校验,客户端不可伪造关键字段(uid/level 由后端反查)。
- 内容安全: 输入 / 输出双向过滤(敏感词 + 内容审核 API);直播业务额外接广电级审核。
- 工具调用最小权限: 写操作需用户/坐席确认。
- 防泄露: System Prompt 不下发到客户端;trace 中敏感字段脱敏。

## 4. 合规

- **数据本地化**: 中国境内用户数据存于境内集群。
- **GDPR / 个保法**:
  - 收集前明示授权(C 端首次进入弹隐私协议;WebView 由宿主负责)。
  - 用户可申请数据导出 / 删除(后台工单流)。
  - 留存策略: 消息默认 180 天,可按租户调整;到期归档。
- **直播/点播专项合规(广电 / 网信办 / 未成年保护法)**:
  - **未成年人打赏退款**: 强制人工 + 双人复核;监护人证明、支付凭证、设备指纹收集合规;7 个工作日内书面答复。
  - **UGC 举报**: 涉黄 / 涉政 / 涉暴 立案后 ≤ 30min 完成处置(截流、封号、留证);所有举报全程留痕,可对接监管反馈接口。
  - **版权**: 收到版权方下架通知 24h 内响应;客服侧支持一键转版权工单。
  - **打赏/虚拟币**: 单笔与日累计阈值告警;主播分成提现合规校验。
  - **直播实名**: 主播咨询场景需校验实名状态。
  - **节目下架原因**: 客服侧不暴露具体审核细节,统一文案。
- **行业**: 金融/医疗租户开启额外加密 + 双人审批配置变更。
- **可审计**: 所有 AI 决策可追溯到 prompt 版本 + 模型 + 命中知识 + 工具调用 + `live_context` 快照。

## 5. 可观测性

- **日志**: 结构化 JSON,统一 `trace_id / session_id / tenant_id`,集中到 ELK / Loki。
- **指标**: Prometheus + Grafana,关键 SLI dashboard。
- **链路追踪**: OpenTelemetry,跨服务 + LLM 调用全链路。
- **AI Trace**: 单独存储,带 prompt / 命中 / token / 成本,支持按会话回放。
- **告警**: P0 电话 / P1 短信 / P2 IM,告警去重 + 抑制。

## 6. 兼容性

- 浏览器: Chrome / Safari / Edge / Firefox 最近 2 版;IE 不支持。
- 系统: iOS 14+ / Android 8+。
- WebView: WKWebView(iOS)、System WebView(Android 60+)。
- 屏幕: 360px ~ 4K。
- 暗色模式: 系统跟随。
- 国际化: 默认中文,预留 i18n(英文)。
- 无障碍: WCAG 2.1 AA。

## 7. 可维护性

- 微服务边界清晰,代码规范 ESLint / Prettier / Checkstyle。
- 单元测试覆盖率 ≥ 70%,关键链路 E2E。
- 公共组件库(C 端 + 后台)统一 Storybook 维护。
- 配置中心化: 业务配置走 Nacos,不能写死代码。
- 文档: API(OpenAPI 自动生成)、架构图、运维手册。

## 8. 可演进性

- C 端 SDK 化: V1 单页,V2 抽出可嵌入 SDK(JS / iOS / Android),复用消息协议。
- 多租户: 数据模型已带 `tenant_id`,V2 开启 SaaS。
- 多渠道: 公众号 / 企微 / 抖音 / 小程序 通过统一 IM 网关接入。
- 多模态: 协议预留 `audio / video / location` 类型。

## 9. 成本

- AI Token 成本占大头: 通过缓存、Haiku 降级、Prompt 精简、RAG 控制 chunk 数控制。
- 长连接成本: 单节点 50k 连接 / 4GB 内存 / 2vCPU,目标月成本 < ¥10/万连接。
- 报表上 ClickHouse 而非 ES 节省 5x+ 存储。
- 冷数据自动归档 OSS 低频。
