## 2. 模块职责

| 服务 | 职责 |
|------|------|
| WS Gateway | 维护客户端长连接、心跳、推送、连接路由 |
| Session | 会话生命周期、状态机、转人工流程编排 |
| Routing | 分配策略、技能组、排队、超时溢出 |
| Agent | 坐席工作台 BFF、坐席状态、并行控制 |
| AI-Hub | 意图/情绪/RAG/Prompt 编排/Tool 调用/降级 |
| Knowledge | 知识库 CRUD、文档解析、切片、向量化、检索调试 |
| Tool | 工具注册与执行,对接直播/点播上下游(播放诊断 / CDN / IM / 礼物 / 订阅 / 内容审核 / 版权 / 风控) |
| LiveCtx | 收集与回放 `live_context`,与播放器 SDK / CDN 监控对接,提供 QoE 合并视图 |
| Notify | 公告、快捷按钮、推送 / WebHook |
| User | C 端用户身份、画像、黑白名单 |
| Tenant | 多租户配额、品牌、SSO |
| Upload | 文件直传 OSS,临时签名,病毒扫描 |
| Audit | 审计日志、操作流水 |
| Report | OLAP 数据 / 实时大屏 / 报表导出 |

## 3. 技术选型

### 3.1 前端

- **框架**: React 18 + TypeScript + Vite 5
- **样式**: Tailwind CSS + CSS Variables 主题、`backdrop-filter` 实现玻璃材质
- **状态**: Zustand(轻)+ TanStack Query(服务态)
- **长连接**: 原生 WebSocket + 自研 reconnecting wrapper(指数退避、消息去重、ack)
- **虚拟列表**: `@tanstack/react-virtual`
- **国际化**: i18next
- **构建**: Vite + 模块联邦(给 SDK 形态预留)
- **测试**: Vitest + Playwright (E2E)

### 3.2 后端

- **语言**:
  - 高并发长连接、网关、消息总线消费 → **Go**(性能、内存)
  - 业务服务、AI 编排 → **Java (Spring Boot 3) 或 Node.js (NestJS)**(团队栈而定;本 PRD 以 Java 为默认)
  - AI 推理与工具:Python(FastAPI)用于模型/向量/数据脚本
- **框架**: Spring Cloud / Spring Cloud Gateway,或者 Go 用 `gin` + `nats`
- **协议**: 内部 gRPC,对外 REST + WS
- **服务治理**: Nacos(注册 + 配置)、Sentinel(限流/熔断)
- **任务**: XXL-Job / Temporal(长流程编排,如对话归档、工单超时)

### 3.3 数据层

- **MySQL 8**: 业务关键事务(用户、坐席、配置、工单)
- **MongoDB**: 消息、会话快照(分片,按 tenant + 时间分库)
- **Redis Cluster**: 会话路由、热点缓存、限流计数、Stream 队列
- **Kafka**: 消息总线、AI 调用流水、报表入湖
- **Elasticsearch**: 会话/消息全文检索
- **向量库**: Milvus 2.x 或 Qdrant(支持过滤、Hybrid)
- **对象存储**: 阿里云 OSS / S3
- **数仓**: ClickHouse(实时报表) + Hive/Iceberg(离线)

### 3.4 AI 层

- **LLM**: 默认 **OpenAI ChatGPT API**(默认模型 `gpt-4o-mini`,可后台切到 `gpt-4o` / `gpt-4.1` / `o4-mini` 等);支持 Azure OpenAI、自建 OpenAI 兼容(vLLM / Ollama / DeepSeek / 通义)、Anthropic(协议适配)。模型 / Key / base_url / 参数全部在管理后台「AI 配置 → 模型与路由」维护,KMS 加密。
- **嵌入**: 默认 OpenAI `text-embedding-3-large`,可切 BGE-M3(自部署)。
- **Reranker**: BGE-Reranker-v2-m3(自部署)。
- **向量库**: Milvus,FAQ 与知识库 chunk 走独立 collection。
- **推理网关**: 自研 LLM Router(以 OpenAI Chat Completions 为统一协议,多家路由 + 健康检查 + 缓存 + 流式 + 限速 + fallback)。
- **不再使用**: 在线意图分类 / 情绪小模型(改为离线 LLM 批跑标注用于报表)。

### 3.5 基础设施

- **容器**: Docker
- **编排**: Kubernetes (生产 ≥ 1.27)
- **网格**: 可选 Istio(流量切分、灰度)
- **入口**: Nginx Ingress + WAF(自建或云 WAF)
- **CDN**: 阿里云 / 腾讯云 / Cloudflare(静态资源 + WS 加速)
- **CI/CD**: GitLab CI / Argo CD,蓝绿 + 金丝雀
- **可观测**: OpenTelemetry → Tempo / Jaeger;Prometheus + Grafana;ELK / Loki

## 4. 部署拓扑

- 多可用区(同城 3 AZ)、跨区域容灾(同 region 主备 + 异地热备)。
- WS Gateway 与后端服务分开伸缩;Gateway 节点数按"长连接数 / 50k"配。
- LLM 推理走专用 GPU 节点池;CPU 业务走普通节点池。
- 数据库主从异地 + binlog 同步到容灾;Redis 多 AZ Sentinel。

## 5. 关键链路时序

### 5.1 用户发消息 → AI 回复 (流式)

```
C 端 ──(WS msg)──► Gateway ──(Kafka: chat.in)──► Session 服务
   ▲                                              │
   │                                              ▼
   │                              拼接历史/画像/RAG 检索
   │                                              │
   │                                              ▼
   │                              AI-Hub 调 LLM(stream)
   │                                              │
   │                                ┌─────流式 token─────┐
   │                                ▼                    ▼
   │                            Session 落库          Gateway 推 C 端
   └──(WS chunks)──────────────────────────────────────────┘
```

### 5.2 转人工

```
AI-Hub 决策 HANDOFF ─► Session 标记排队 ─► Routing 分配
   ─► Gateway 推坐席 (新会话)
   ─► 坐席接受 ─► 双向 WS 转发(Gateway 处理)
```

### 5.3 播放问题 + 直播上下文

```
C 端 (悬浮入口) ─► JSBridge.setLiveContext + 一键播放诊断
   ─► 客服 H5 携 live_context 进会话
   ─► AI-Hub 拼 Prompt + 调 get_play_diagnostics(QoE) + get_room_info
   ─► 判定: 客户端网络问题 → 建议切清晰度 (switch_quality_hint)
            | CDN/源站问题 → 转「播放与技术」+ 自动建议公告
            | 鉴权/风控   → 转对应队列
```

## 6. 上下游业务集成边界

| 系统 | 协议/形态 | 关键点 |
|------|----------|--------|
| 播放器 SDK(iOS/Android/Web) | JSBridge + 上报 SDK | 提供 ring buffer 60s 日志、清晰度档位、卡顿事件;协议字段版本化 |
| CDN(阿里/腾讯/快手云等) | 厂商 OpenAPI + 自建 QoE 上报 | 节点状态、5xx 比例、节点路由;故障 webhook 触发自动公告 |
| 直播 IM / 弹幕 | gRPC / WebSocket | 直播间快照、禁言 |
| 礼物 / 虚拟币 | REST | 流水、退款 |
| 订阅 / 支付 | REST + 异步回调 | 续费、退款、未成年退款工单 |
| 内容审核 | REST + 流式 | 输入/输出敏感词与色情/暴恐识别 |
| 版权系统 | REST + 工单回流 | 节目下架、版权方反馈 |
| 风控 | REST | 设备指纹、未成年识别、黑名单 |
| 监控/数仓 | Kafka / OpenTelemetry | QoE 数据落湖,反哺 AI 训练 |
