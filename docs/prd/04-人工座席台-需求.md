# 人工座席工作台 · 产品需求

供 **客服坐席 / 主管 / 质检** 使用的 Web 应用,目标是「单坐席并行 5~8 路会话仍高效」。

## 1. 角色与权限

| 角色 | 权限要点 |
|------|---------|
| 普通坐席 | 接入会话、回复、调用工具、转接、置忙、查看自己历史 |
| 高级坐席 | + 跨技能组接入、+ 知识库建议提交 |
| 班组长 | + 监听/插话/抢接、+ 强制转接、+ 班组报表 |
| 质检 | 只读会话 + 打分/标注 |

## 2. 工作台布局

```
┌─────────────────────────────────────────────────────────────┐
│ 顶部: 状态(空闲/忙碌/小休) | 通知 | 工号 | 设置             │
├──────────┬──────────────────────────────────┬────────────────┤
│ 左:会话列表│       中: 当前会话消息流           │ 右: 上下文与工具│
│ - 待接入   │   (含 AI 历史 + 转接摘要)          │ - 用户画像      │
│ - 进行中   │                                  │ - 订单/工单     │
│ - 历史     │                                  │ - 知识库搜索    │
│           │                                  │ - 建议回复(AI) │
├──────────┴──────────────────────────────────┴────────────────┤
│ 输入区: 富文本+附件+快捷短语+一键应答(AI 草稿)               │
└──────────────────────────────────────────────────────────────┘
```

## 3. 核心功能

### 3.1 会话接入

- 自动派单(默认):新会话弹出 Toast + 振铃,8s 内未接受自动重派。
- 手动抢单(支持):待接入列表中,主管可开关。
- 同时进行: 默认 5 路,主管可调,达到上限自动置满。
- 接入即看到:对话历史(含 AI)、Handoff Packet 摘要、用户画像、关联订单。

### 3.2 消息回复

- 富文本: Markdown 子集 + 表情包 + 图片/文件。
- 快捷短语: 个人/班组/全局三层,支持变量(`{用户姓名}`、`{订单号}`)、热键(`/退货政策`)。
- AI 建议回复:
  - 默认基于当前对话与知识库,实时生成 1~3 条候选,坐席一键采纳/编辑后发送。
  - 也可主动调用 `Ctrl+/` 让 AI 重新生成。
- 草稿: 输入未发送时自动保存,刷新不丢。
- 撤回: 2 分钟内可撤回。
- 输入提示: 用户端可见"客服正在输入..."。

### 3.3 协作能力

- **内部备注**: 仅坐席侧可见的会话备注/标签。
- **转接**:
  - 转人工(指定坐席/技能组),携带摘要。
  - 转 AI(让 AI 接管,如夜班托管)。
- **求助**: 邀请主管"监听 / 插话 / 抢接"。
- **会话合并/拆分**: 同一用户多渠道会话可合并视图。

### 3.4 工具与上下文(直播/点播专用)

右侧 Tab(每个 Tab 都基于当前 `live_context` 自动加载,坐席接入即可见):

- **播放诊断**: 服务端 QoE + 客户端 60s 日志 + 卡顿热力图;一键"建议切清晰度 / 重进直播间 / 推送公告"。
- **直播间快照**: 节目封面、当前主播、在线人数、清晰度档位、CDN 节点健康分;支持"加入观察"实时跟看几秒。
- **节目(点播)**: 版权方、上下架、字幕状态、是否会员专享、最近相同问题工单。
- **会员与订阅**: 等级、到期、连续包月、家庭组、近期订阅/续费/打赏流水;一键发起退款(写操作走二次审批)。
- **主播档案**: 签约状态、违规记录、最近开播、打赏统计(主播咨询时显示)。
- **举报与合规**: 当前会话关联的举报记录、证据片段、风控判定;支持一键创建合规工单。
- **历史会话**: 同一 UID 跨直播间的历史。
- **知识库搜索**: 输入即搜,可一键引用到回复;命中后还能"+ 反馈不准"提交训练队列。
- **AI 建议**: 基于上述全部上下文实时生成 1~3 条候选。
- **内部备注**: 仅坐席侧可见。

工具操作(切清晰度建议、退款、封禁、推送公告)全部审计;高风险操作(退款 ≥ 500、未成年退款、主播封禁)需主管二次审批。

### 3.5 工单 (V1 仅预留)

- 「转工单」按钮:把当前会话与上下文打包成工单,跳转工单系统(外部或后续自建)。

### 3.6 满意度与结束

- 结束会话需:选择会话标签(多选,如"已解决/咨询类/物流问题")+ 简要小结(选填)。
- 结束后向用户推送评价卡(1-5 星 + 标签 + 备注)。
- 评价回流到坐席个人 KPI 页面。

## 4. 状态机与考勤

- 状态: `离线 / 空闲 / 忙碌 / 暂离(用餐/培训/小休) / 小休结束待返`。
- 暂离需选原因 + 主管可设最长时长。
- 系统自动统计在线时长、忙时长、平均响应、平均处理、并行峰值。

## 5. 主管视图

- **实时监控大屏**:
  - 排队人数、平均等待、超时排队、各技能组负载
  - 各坐席状态/并行数/响应时长
  - AI 拦截率、转人工率(实时)
  - **直播/点播业务大盘**: TOP N 故障直播间(同一 `room_id` 进客服 ≥ 阈值)、CDN 节点 5xx 排行、节目下架投诉量、未成年退款实时件数
  - **赛事/大主播预警**: 与运营日历联动,赛事开播前 30 分钟自动提示扩容、临时增援
- **干预**:
  - 监听任意会话(用户与坐席无感知)
  - 插话(坐席侧标记主管发言)
  - 抢接(强制转移会话)
  - 强制下线 / 强制结束
- **排班**: 周/月排班表,支持导入,异常告警(缺岗、迟到、超时)。

## 6. 质检

- 抽样规则: 比例 / 关键词 / 低分会话。
- 评分卡(可配置维度): 礼貌、准确、效率、合规等;每项权重。
- 申诉: 坐席对质检结果可申诉。
- 与 AI 训练联动: 质检不合格回复可标记为反例进入训练。

## 7. 技术与体验要点

- React + TS + 长连接(WS),与 C 端共用消息协议。
- 多会话切换不丢草稿、不重复加载历史(虚拟列表 + 缓存)。
- 桌面通知 + 声音 + Tab 标题闪烁(`(3) 待接入会话`)。
- 网络断开:顶部红条提醒,期间会话冻结,恢复后增量同步。
- 浏览器多 Tab 互斥: 同一坐席只能在一处接入,后开 Tab 提示。
