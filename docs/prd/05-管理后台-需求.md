# 管理后台 · 产品需求

供 **管理员 / 运营 / 训练师** 使用,集中配置业务规则、知识库、UI 文案、人员等。

## 1. 模块总览

| 模块 | 子模块 |
|------|--------|
| 工作台 | 概览看板、待办 |
| 接入与渠道 | 站点接入、JS SDK 配置、WebView 协议、`live_context` 协议、域名白名单 |
| C 端配置 | 公告(支持按直播间/节目定向)、快捷按钮、**常见问题(多级)**、欢迎语(按场景)、主题、敏感词 |
| AI 配置 | 知识库、Prompt 模板(按场景)、模型与路由(默认 ChatGPT)、工具、A/B 实验 |
| 客服管理 | 坐席、技能组(播放/会员/内容/主播/未成年/VIP)、排班、分配策略 |
| 业务集成 | 播放诊断 API、CDN 状态源、IM 弹幕、礼物/虚拟币、订阅/支付、内容审核、版权系统 对接 |
| 会话与工单 | 全部会话(按 room/vod 检索)、转人工、未成年退款审批、举报立案、工单(预留) |
| 报表 | 实时大屏、运营报表、AI 报表、坐席绩效、**业务报表(按节目/直播间/CDN/设备)** |
| 系统 | 角色权限、审计日志、租户、计费(预留) |

## 2. C 端配置(本期重点)

### 2.1 公告管理

- 列表字段: 标题、级别(info/warning/critical)、内容、链接、生效时间、状态、生效站点/渠道。
- 支持富文本(限定子集)、Emoji。
- 多条公告: 优先级 + 轮播 / critical 常驻。
- **预览**:右侧实时预览跑马灯效果(PC + Mobile)。
- 立即下发: 保存后通过推送通道实时刷新所有在线 C 端。
- **定向投放(直播/点播)**:
  - 维度: 用户标签 / 百分比 / 站点 / **`room_id` / `vod_id` / `anchor_id` / `cdn_node`**
  - 例: "直播间 8001 故障公告" 只对该 room_id 内进客服的用户展示
  - 与运维监控联动: CDN 节点 5xx > 阈值时自动建议生成抢修公告(待管理员一键确认上线)

### 2.2 快捷按钮管理

- 字段: `label`、`payload`、场景(欢迎/售前/售后/通用)、图标、排序、用户标签。
- 支持模板变量、表情。
- 分组: 同一场景多按钮,按 `sortOrder` 展示。
- 预览: 输入框上方真实样式预览。
- 灰度: 同公告。
- **数据面**: 每个按钮的点击率、后续是否转人工、是否被 AI 解决。

### 2.3 常见问题(多级 FAQ)

C 端在欢迎语之后展示的多级目录式 FAQ,目标拦截高频简单问题、零 Token 消耗。

- **结构**: 树形,任意层级(推荐 ≤ 3 级);每个节点 = 分类节点 或 叶子(QA)。
- **节点字段**:
  - 通用: `id, title, icon, sort_order, status(草稿/上线/下线), target_segment(用户标签灰度), valid_range(生效时间)`
  - 分类节点: `children[]`
  - 叶子节点: `answer.content_md`(富文本,支持图片/链接/Markdown 子集)、`attachments[]`、`follow_ups[]`(关联其它叶子)、`actions[]`(send_text / handoff / open_link / open_form)
- **管理操作**:
  - 树形拖拽排序、复制粘贴节点、批量导入(CSV / JSON)、批量导出。
  - 单节点版本历史 + 回滚。
  - 同义问列表(用于"相似匹配"命中,阈值后台可配,默认 0.86)。
  - 全局搜索(按标题、答案内容、ID)。
- **预览**: 内嵌 C 端模拟器,真实展示首层网格 / 下钻 / 命中答案的过程。
- **灰度上线**: 按用户标签 / 百分比 / 站点;支持「先发草稿 → 灰度 → 全量」三段。
- **数据面**:
  - 节点曝光、点击率、命中后是否触发 LLM 兜底、是否最终转人工、用户评价正负比。
  - 「未命中 Top N」:统计被用户问到却没有任何 FAQ 命中的问题文本,引导新增/优化。
  - 「低效叶子 Top N」:命中后仍转人工率 > 阈值的叶子,优先优化文案。
- **变更生效**: 保存上线后通过 `event.faq_update` 推送给所有在线 C 端,5s 内全量更新。

### 2.4 欢迎语 / 结束语 / 满意度问卷

- 多套话术,根据场景/用户标签自动选择。
- 满意度问卷:维度可配,标签库可配,匿名开关。

### 2.5 主题与品牌

- Logo、主色、明暗模式默认值、玻璃材质强度。

### 2.7 业务集成(直播/点播专属)

集中管理与上下游业务系统的对接,所有 AI 工具与坐席操作都依赖这里:

| 集成 | 用途 | 关键配置 |
|------|------|----------|
| 播放诊断 API | `get_play_diagnostics` 数据源 | endpoint、鉴权、字段映射、超时 |
| CDN 状态源 | 实时拉 CDN 节点 5xx / RTT,驱动公告与故障定位 | 多 CDN 厂商接入(阿里/腾讯/快手云) |
| 播放器 SDK 协议 | JSBridge `setLiveContext` / `requestPlayDiagnostics` 字段约定 | 版本兼容矩阵 |
| IM / 弹幕 | 直播间快照、踢人禁言 | 协议适配 |
| 礼物 / 虚拟币 | 打赏流水、退款 | 退款路径 + 风控阈值 |
| 订阅 / 支付 | 会员、连续包月、退款 | 多支付通道、未成年特殊路径 |
| 内容审核 | 输出 + UGC 输入审核 | 接入家数与 fallback |
| 版权系统 | 节目下架、版权方对接 | 工单回流 |
| 风控系统 | 黑名单、设备指纹、未成年识别 | 字段同步 |

每项集成有: endpoint、密钥(KMS)、超时、降级策略、健康分;对外暴露 `工具调试器`(模拟参数 → 看真实返回)。

### 2.8 敏感词与黑名单

- 自建词库 + 第三方内容安全 API 双重校验。
- 黑名单用户(IP / 设备 / 用户 ID)直接拒绝接入或转人工。

## 3. AI 配置

### 3.1 知识库

- 多知识库(按类目/技能组/语言)。
- 文档管理: 上传(PDF/Word/Markdown/网页抓取)、解析、切片预览、版本、生效区间。
- QA 对管理: 问 - 答 - 同义问 - 标签 - 引用文档。
- 训练队列: 来自坐席 / 训练师 / 用户反馈的待标注条目。
- 检索调试: 输入 query → 看召回链路(向量分、BM25 分、Rerank 分、最终命中)。

### 3.2 Prompt 模板

- 模板版本化、A/B、按场景路由。
- 变量字典(用户、订单、知识、摘要)。
- 上线前强制跑「回归测试集」。

### 3.3 工具(Function)

- 注册/编辑工具: 名称、描述、入参 schema、出参示例、超时、权限、是否需要二次确认。
- 模拟调试: 输入参数 → 看真实调用结果。

### 3.4 模型与路由

- **默认 ChatGPT**: 系统出厂即预置一档 `provider=openai`、`base_url=https://api.openai.com/v1`、`model=gpt-4o-mini` 的「default」档,只需填入 API Key 即可使用。
- **模型档位 CRUD**:
  - 字段: 名称、provider(openai / azure_openai / anthropic / 自建 OpenAI 兼容)、base_url、api_key(密文,UI 仅展示后 4 位,可点击"重新填写")、model 名称、temperature/top_p/max_tokens/penalty 等参数、stream 开关、超时、RPM/TPM 限速、fallback 档位。
  - **API Key 配置**: 通过表单填入,后端用 KMS 加密存储,前端永不回显;支持"测试连接"按钮(调用一次最小 chat 验证)。
  - **模型选择**: `model` 字段为自由文本,跟随 OpenAI 官方型号(gpt-4o / gpt-4o-mini / gpt-4.1 / o3 / o4-mini ...);Azure 填 deployment 名称;自建 OpenAI 兼容填实际模型 ID。
- **路由策略**: 按场景(在线对话 / 嵌入 / 重排 / 试聊) + 按租户 + 按 RAG 命中难度 + 按延迟成功率 + 按成本,支持优先级与权重。
- **健康监控**: 各档位实时健康分(成功率、P95 延迟、可用性),失败自动切 fallback 并告警。
- **配额与计费**: 每租户 / 每会话 Token、QPS;月度 Token 预算 + 80%/100% 阈值。
- **计费看板**: 按档位 / 模型 / 租户 / 场景的 Token 与费用走势,导出 CSV。

### 3.5 转人工策略

- 关键词、RAG 置信阈值、最大未解决轮次、白/黑名单、用户主动按钮 — 全部可视化配置。
- FAQ 相似匹配阈值(默认 0.86)。
- 模拟器: 录入一段对话,看决策器输出(命中 FAQ / RAG / 工具 / LLM / 转人工 哪条路径)。

## 4. 客服管理

- 坐席 CRUD、批量导入、技能组、上下班时间、最大并行数。
- 技能组: 名称、关联类目、优先级、溢出规则。
- 分配策略: 选择/组合策略,按技能组覆盖。
- 排班: 周视图,支持模板、复制、冲突检测、缺岗预警。

## 5. 会话与工单

- 会话搜索: 时间、用户、坐席、关键词、是否转人工、满意度。
- 详情: 完整对话(AI/坐席/系统)+ trace + 工具调用记录 + 评价。
- 导出: CSV / 离线脱敏。
- 工单: 列表 + 状态流转(预留接口)。

## 6. 报表

### 6.1 实时大屏

- 在线 C 端、活跃会话、排队、AI 拦截率、首响、坐席状态分布。
- 1s ~ 5s 刷新,WebSocket 推送。

### 6.2 运营报表

- 维度: 时间、渠道、技能组、坐席、问题分类、**直播间(room_id)/ 节目(vod_id)/ 主播(anchor_id)/ CDN 节点 / 设备 / 网络类型**。
- 指标: 会话量、AI 拦截率、转人工率、首响、平均处理、满意度、放弃率、退款金额、未成年退款件数、举报立案数。
- 下钻: 任意单元格点击进会话列表;按直播间下钻可看"该直播间客服热度曲线 vs 在线人数曲线"。
- 故障报表: TOP N 故障直播间 / CDN 节点;复发问题节目;字幕错误节目。

### 6.3 AI 报表

- Token / 成本、命中率、回退率、A/B 对比、知识库点击率、低分 Top N 问题。

### 6.4 坐席绩效

- 在线时长、接入量、并行数、平均响应、满意度、质检分。
- 班组 PK、个人榜单。

## 7. 系统

- RBAC(角色 + 资源 + 操作),支持自定义角色。
- 审计日志: 配置改动、敏感操作全记录。
- 租户(预留): 数据隔离、配额、品牌定制。
- 单点登录(SSO,OIDC/SAML)、双因素。

## 8. 体验

- 中后台沿用现代极简风格(非 Glassmorphism),采用清晰栅格 + 数据密集表格。
- 关键操作二次确认 + 操作回执。
- 所有列表支持快捷过滤、列自定义、CSV 导出。
- 所有"上线"操作均有 dry-run 预览或灰度选项。
