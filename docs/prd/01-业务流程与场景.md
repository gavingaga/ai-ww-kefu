## 2. 转人工触发策略

满足任一即触发,优先级从高到低:

1. **强制规则**: 输入命中关键词(投诉、退款、人工、未成年、监管、版权、举报),或多轮重复未解决(连续 ≥ 2 次"没解决我的问题")。
2. **业务必转**:
   - 退款 / 取消订阅(尤其涉及未成年人打赏)
   - UGC 举报(涉政 / 涉黄 / 涉暴)
   - 版权下架投诉
   - 主播账号封禁申诉
   - 大额虚拟币/礼物纠纷
3. **播放诊断后业务侧故障**: AI 调用 `get_play_diagnostics` 判定为 CDN/源站故障 / 限流 / 鉴权失败 → 自动转技术支持队列,且首条消息附带诊断卡片。
4. **AI 置信度**: RAG 召回 top-1 相似度 < 阈值(默认 0.72);LLM 自陈"无法回答"。
5. **业务白/黑名单**: VIP 会员、签约主播、历史投诉用户走专席队列。
6. **主动按钮**: 输入框 / AI 回复气泡 / FAQ 答案动作区 / 播放器悬浮菜单内的"转人工"。
7. **超时兜底**: AI 连续 3 轮未解决,自动提示并引导转人工。

转人工时:

- 系统将最近 N 轮对话、用户画像、订单上下文打包成 **会话摘要**(由 LLM 生成),展示在坐席侧边栏。
- 排队期间用户侧显示"前面还有 X 位,预计等待 Y 秒",AI 仍可继续答疑。

## 3. 排队与分配

- **技能组路由**(直播/点播业务):
  - **播放与技术**: 卡顿、黑屏、无声、清晰度、投屏、CDN/鉴权问题
  - **会员与支付**: 订阅、续费、退款、连续包月、家庭组
  - **内容与版权**: 节目下架、字幕错误、UGC 举报、版权方对接
  - **主播与社区**: 主播咨询、礼物/虚拟币、封禁申诉、社区规范
  - **未成年合规**: 未成年人打赏退款专席(必须人工 + 双人复核)
  - **VIP 专席**: 高等级会员 / 签约主播
- 据会话标签 + `live_context.scene` 自动匹配。
- **分配策略**(可配置组合):
  - 最少会话优先(默认)
  - 熟客优先(同一用户优先分配历史坐席)
  - 优先级(VIP 走专席队列)
  - 轮询 / 随机
- **超时升级**: 排队 > 60s 自动溢出到上级技能组;> 180s 触发主管告警。

## 4. 坐席状态机

```
离线 ─登录─► 空闲 ─接入会话─► 忙碌 ─结束会话─► 空闲
                  │                    │
                  └──置忙/小休/培训───► 暂离
```

## 5. 典型场景

### 场景 A: 直播间卡顿(AI 一键解决)
用户在直播间内点客服悬浮入口 → C 端自动注入 `live_context = { room_id: 8001, anchor: "xxx", quality: "1080p", cdn_node: "sh-3", buffer_events: 4次/60s }` + 上一条用户输入"画面好卡" → AI 调 `get_play_diagnostics` → 判定为本地网络抖动(下行 1.2Mbps) → 回复"建议切到流畅清晰度,并附一键切换按钮" → 用户点击切换 → AI 后续 30s 监测无卡顿 → 自动结束并请求评价。

### 场景 A2: 多级 FAQ 命中(零 LLM)
用户进入会话 → 欢迎语后展示「常见问题」 → 点击「会员与订阅 › 怎么取消连续包月?」→ 系统直接回复操作步骤 + 「打开订阅管理」「转人工」按钮 → 用户操作完成,点"已解决"。该路径不消耗 LLM Token。

### 场景 B: CDN 故障(AI → 技术支持)
多用户在同一直播间报"打不开",AI 调诊断接口发现该直播间 `cdn_node = sh-3` 5xx 率 > 30% → 命中"业务侧故障"规则 → 自动转「播放与技术」队列 + 推送诊断卡片;同时通知运营在 C 端公告跑马灯增加"直播间 8001 正在抢修"红色 critical 条目,后续同房间用户进客服直接看到。

### 场景 C: 未成年人打赏退款(强制人工 + 合规)
用户输入"小孩偷用我手机充了 5000 元礼物,想退" → 命中"未成年"关键词 → 立即转「未成年合规」专席,**全程禁用 AI 自动回复** → 坐席侧栏显示订单流水、设备指纹、近 30 天打赏记录,触发双人复核流程。

### 场景 D: UGC 举报
用户在直播间举报涉黄,跳转客服 → C 端自动注入 `live_context.report = { type: "porn", evidence_clip_url: "...", room_id, anchor_id }` → 命中合规规则,直接转「内容与版权」 → 坐席接入后一键调"截流/封号/工单创建",留痕到合规系统。

### 场景 E: 高峰排队(赛事直播)
赛事开播 5 分钟内会话量 10x 飙升 → 自动扩容触发 → 排队期间 AI 持续答疑(节目时间表、清晰度建议、回看/录像提示) → 坐席空闲后自动接入,排队页显示"赛事高峰期,前面还有 X 位"。

### 场景 F: 主播封禁申诉
主播账号被封 → 通过主播端入口进客服 → 自动注入 `live_context.role = "anchor"` + 封禁原因码 → 路由到「主播与社区」专席 → 坐席侧栏直接显示封禁记录、违规截图、申诉模板。

## 6. 异常与降级

| 异常 | 策略 |
|------|------|
| AI 服务不可用 | 自动切换到"全人工模式",顶部 Banner 提示 |
| 知识库不可用 | LLM 走通用模型回答 + 提示"信息可能不准,可转人工" |
| 长连接断开 | 客户端指数退避重连(1s/2s/4s/...max 30s),拉取断连期间消息 |
| 坐席侧崩溃 | 会话自动转回排队,30s 内由其他坐席接续,上下文不丢 |
| 播放诊断 API 不可用 | 跳过自动诊断,提示用户上传截图/录屏,人工兜底 |
| CDN 大面积故障 | 客服系统不阻塞,公告跑马灯自动挂"系统抢修中"提示,FAQ 卡片置顶"播放故障公告" |
| 直播突发流量 | 自动扩容 Gateway / Routing,排队水位 > 阈值时启用"AI 陪聊 + 工单异步回复"模式 |
