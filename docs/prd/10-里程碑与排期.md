# 里程碑与排期

> 团队规模假设:前端 4(C 端 2 + 后台/座席 2)、后端 6、AI 2、测试 2、设计 1、PM 1。
> 起点假设 2026-05-01,以下周次为相对周。

## M0 · 准备 (W1)
- 立项、技术选型确认、环境与 CI/CD 搭建。
- 完成视觉规范走查 (Figma) 与 Design Token。
- API/协议第一版评审 (OpenAPI + WS 协议)。

## M1 · 基础骨架 (W2 ~ W4)
- WS Gateway MVP:连接 / 心跳 / 路由 / Redis 映射。
- Session 服务:会话创建、消息收发、MongoDB 落库。
- C 端骨架:登录、消息流、文本收发、虚拟列表、玻璃材质组件库。
- 后台骨架:登录 / 权限 / 公告 + 快捷按钮 配置。
- DevOps:K8s + 监控 + 日志 + 灰度发布通道。
- **里程碑产出**: 内部联调可文本聊天,公告/快捷按钮端到端可用。

## M2 · AI 中枢接入 (W5 ~ W7)
- AI-Hub:RAG(向量+BM25+Rerank)、Prompt 编排、流式输出。
- 知识库:文档上传 / 切片 / 入库 / 检索调试。
- 决策器 + 转人工策略(规则 / 阈值 / 关键词;含未成年/举报/版权 必转规则)。
- LLM Router:默认 ChatGPT (`gpt-4o-mini`),Key/模型/路由后台可配。
- 工具调用框架 + 直播/点播领域工具:`get_play_diagnostics` / `get_room_info` / `get_membership` 等。
- `live_context` 注入: JSBridge + Prompt 拼装 + 安全校验。
- C 端 AI 思考动效、流式 token、AI 引用展示。
- **里程碑产出**: AI 拦截链路全通,带上下文的"卡顿建议"准确率达基线。

## M2.5 · 业务集成 (W6 ~ W8,与 M2/M3 并行)
- 播放诊断 API、CDN 状态源、订阅/支付、IM 弹幕、礼物/虚拟币、内容审核、版权、风控 — 全部对接联调。
- 工具调试器、健康分、降级策略上线。
- 故障公告自动建议(CDN 5xx → 草稿)。
- **里程碑产出**: AI 与坐席工具调用 SLA 达标。

## M3 · 人工座席台 (W7 ~ W9,与 M2 部分并行)
- 坐席工作台 MVP:接入、消息、上下文、快捷短语、AI 建议回复。
- Routing 服务:技能组 / 分配策略 / 排队 / 超时溢出。
- Handoff Packet 全字段;转人工无缝切换。
- 主管视图:实时大屏 + 监听/插话/抢接。
- **里程碑产出**: 转人工链路全通,坐席并行 5 路稳定。

## M4 · 完整 C 端体验 (W9 ~ W10)
- 图片/文件上传(直传 OSS、压缩、进度)。
- 公告跑马灯多条/灰度/critical 常驻 + 按 `room_id`/`vod_id` 定向。
- 快捷按钮分组/灰度/数据回流。
- 多级常见问题(树形 + 同义问 + 数据面)。
- **直播间悬浮入口 / 半屏抽屉 / 全屏横屏 / PIP 共存**。
- **一键播放诊断卡片**与 SDK 联调。
- 满意度评价、消息撤回、断线重连、未读提示。
- 暗色模式、WebView JSBridge、无障碍。
- **里程碑产出**: C 端达 PRD 验收标准,在主流直播场景内体验通畅。

## M5 · 报表与训练闭环 (W10 ~ W12)
- 实时大屏 / 运营报表 / AI 报表 / 坐席绩效。
- 质检模块 + 训练队列 + 知识库回归测试集。
- A/B 实验、Prompt 版本化。
- **里程碑产出**: 数据驱动闭环成立。

## M6 · 高可用与压测 (W12 ~ W14)
- 多 AZ 部署、跨 region 容灾演练。
- Chaos 演练、性能压测达成 V1 容量 1.5x。
- **直播突发流量演练**: 大主播开播 / 赛事直播 10x~20x 模拟,验证扩容、削峰、降级开关。
- 限流 / 风控 / WAF 调优。
- 安全渗透测试 + 合规审计(未成年退款流程、举报留痕)。
- **里程碑产出**: 通过容量与稳定性验收,SLA 达标,合规审计通过。

## M7 · 灰度上线 (W15)
- 内部 dogfooding(2 周已执行,持续)。
- 邀请 2~3 个真实业务方接入,按用户 % 灰度。
- 监控 / 告警 / on-call 机制建立。

## M8 · GA (W16)
- 全量上线、文档与培训交付。
- 复盘指标:AI 拦截率、首响、满意度、SLA。

## 关键风险与缓解

| 风险 | 缓解 |
|------|------|
| AI 拦截率不达标 | 提前 1 周采集真实语料、上 Hybrid 检索、FAQ 多级树覆盖头部、招训练师驻场 |
| 长连接稳定性 | 提前压测 + Chaos;预备 SSE 降级 |
| LLM 成本超预期 | 切便宜档(`gpt-4o-mini` / 自建)+ 缓存 + Token 预算告警;高复杂走 `gpt-4o` |
| WebView 兼容差异 | 与宿主 App 提前定义 JSBridge + `live_context` 字段版本,准备 polyfill |
| 播放诊断 API 不稳 | 客户端 60s 日志兜底;无 QoE 时引导用户上传截图 |
| 大主播 / 赛事突发流量 | 与运营日历联动预热 + 群发模式 + 工单兜底,提前演练 |
| 未成年退款 / 举报合规 | 流程评审到位、双人复核工具上线、季度合规审计 |
| 业务方接入排期不齐 | 灰度按业务方独立,互不阻塞 |

## 资源与依赖

- LLM 供应商账号(OpenAI 主、Azure / 自建 / 国产备)及配额。
- 内容安全 API、风控、CDN、OSS。
- **业务方系统**: 播放诊断 / CDN 监控 / IM 弹幕 / 礼物虚拟币 / 订阅支付 / 内容审核 / 版权 / 风控 / 用户中心(Tool 对接)。
- **播放器 SDK 团队**: JSBridge 字段、ring buffer 60s 日志、清晰度切换、横屏与 PIP 适配。
- **运营 / 法务 / 安全 / 合规**: 公告模板、隐私协议、未成年退款 SOP、UGC 举报 SOP、版权下架 SOP、广电级内容审核合作。
